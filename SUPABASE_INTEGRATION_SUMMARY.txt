🔑 ENVIRONMENT VARIABLES NEEDED
═══════════════════════════════════════════════════════════════════════════

Add to .env.local:

  # Supabase URLs and Keys
  NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
  NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
  SUPABASE_SERVICE_ROLE_KEY=your-service-role-key-here
  
  # NextAuth
  NEXTAUTH_URL=http://localhost:3000
  NEXTAUTH_SECRET=your-secret-key-min-32-chars
  
  # Optional
  NODE_ENV=development

⚠️  IMPORTANT:
  • Never commit .env.local with real keys
  • Never commit SUPABASE_SERVICE_ROLE_KEY
  • Use separate projects for dev/prod
  • Rotate keys regularly

🗄️  DATABASE SCHEMA
═══════════════════════════════════════════════════════════════════════════

users table:
  id              TEXT PRIMARY KEY
  email           VARCHAR UNIQUE NOT NULL
  name            VARCHAR
  password_hash   VARCHAR NOT NULL
  role            VARCHAR (owner|admin|user)
  status          VARCHAR (active|inactive)
  created_at      TIMESTAMP
  updated_at      TIMESTAMP

Indexes:
  • idx_users_email (for login lookups)
  • idx_users_role (for role queries)

Security:
  • Row Level Security (RLS) enabled
  • Policies for user/admin access
  • Foreign key constraints

🆘 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

Problem: "Cannot connect to Supabase"
Solution: 
  • Check NEXT_PUBLIC_SUPABASE_URL in .env.local
  • Verify project is active in Supabase
  • Test connection with supabase-js client

Problem: "Users already exist" (API seed)
Solution:
  • This is expected - users already seeded
  • Try seeding different user or drop table first
  • Check Supabase table editor

Problem: "Service role key not found"
Solution:
  • Add SUPABASE_SERVICE_ROLE_KEY to .env.local
  • Get it from Supabase Settings → API Keys
  • Restart dev server

Problem: "Invalid password" on login
Solution:
  • Verify password is "password123"
  • Check bcryptjs hash is correct
  • Verify password_hash field exists in table
